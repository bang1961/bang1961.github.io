문제

Radiology GPT — 다국어 판독문 해석 및 질의응답

MULTILINGUAL LLM · 2023–2025
1인 개발
문제
한국어·영어·인도네시아어로 작성된 방사선 판독문은 비전문가가 이해하기 어렵고, 질의응답이 불가능했습니다.
접근
GPT-4로 MIMIC 판독문 기반 Q&A 데이터를 생성하고, LLaMA를 언어별로 LoRA(8bit) 파인튜닝했습니다. 보고서의 핵심 임상 정보를 엔티티-관계 단위로 구조화해 답변이 근거 문장과 매핑되도록 설계했습니다.
결과
응답 일관성 89.5%, 사용자 이해도 4.3/5를 확보했습니다. 번역 손실·코드스위칭은 프롬프트 규칙 강화와 용어 정규화로 완화했습니다.
응답 일관성 89.5%이해도 4.3/5

실시간 STT 서빙 고도화 & VAD 벤치마크

SPEECH AI · 2025–현재
1인 개발
문제
8k 콜센터 환경에서 VAD 품질이 떨어지면 발화 누락과 환청성 텍스트(오탐)가 늘고, 실시간 서빙 구조가 불안정해졌습니다.
접근
WebRTC·pyannote·SpeechBrain VAD를 동일 ASR(ESPNet ha8k) 조건에서 비교(WER/CER, 세그 길이, 에너지 분포 분석)하고, asyncio 없는 multiprocess-only gRPC 서버로 재구성했습니다. 웹 마이크 기반 E2E 테스트 환경을 구축해 콜봇 연동까지 검증했습니다.
결과
pyannote+ESPNet 조합이 가장 우수했고, 저에너지 구간 제거로 오탐을 줄였습니다. 검증된 VAD·STT 구성과 파라미터를 실제 서비스에 적용했습니다.
WER/CER 정량 비교오탐 감소
gRPCESPNetWebRTC/pyannote/SpeechBrain VADDocker

실시간 판독문 오타 교정 시스템

NLP PRODUCT · 2023–2025
1인 개발
문제
의료 보고서 작성 중 발생하는 오타가 판독 품질과 작성 속도에 영향을 주고 있었습니다.
접근
KoSymSpell/SymSpell과 LLM 기반 교정 제안을 결합해, Flask와 JavaScript로 입력 중인 단어를 실시간 분석하고 추천 팝업을 제공하는 모듈을 구현했습니다.
결과
교정 정확도 F1 84.5%를 기록했고, 작성 시간을 약 18% 단축했습니다. 짧은 단어의 과교정 문제는 편집거리 제어로 해결했습니다.
